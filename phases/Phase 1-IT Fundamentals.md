# Phase 1 — IT Fundamentals

Before learning cybersecurity, you must understand how IT systems work.

SOC analysts investigate systems, users, devices, and network activity every day. Without IT fundamentals, security investigations become difficult.

---

# What You Need To Learn

## 1. Computer Basics

Understand:
- How computers work
- CPU, RAM, Disk
- Processes & Services
- File systems
- Applications
- System performance basics

Learn:
- What is a process?
- What is memory usage?
- What is a service?
- What is a task scheduler?

Why It Matters:
- Malware runs as processes
- Attackers abuse services
- SOC analysts investigate process activity

---

## 2. Operating Systems Basics

Learn:
- Windows basics
- Linux basics
- File paths
- Permissions
- User accounts
- Administrative privileges

Understand:
- Difference between user/admin
- How applications run
- Startup locations
- Logs

Why It Matters:
- Most attacks target operating systems

---

## 3. Command Line Basics

### Windows Commands
Learn:
- ipconfig
- netstat
- tasklist
- whoami
- nslookup
- ping

### Linux Commands
Learn:
- ls
- cd
- pwd
- ps
- grep
- top
- chmod
- curl

Why It Matters:
- SOC analysts investigate systems through command line frequently

---

## 4. File Systems

Learn:
- Windows file structure
- Linux file structure
- Hidden files
- Temporary folders
- Startup folders

Important Locations:
### Windows
- C:\Windows
- AppData
- Temp
- Startup folders

### Linux
- /etc
- /var/log
- /tmp
- /home

Why It Matters:
- Malware often hides in these locations

---

## 5. Processes & Services

Learn:
- What is a process?
- Parent-child process relationships
- Background services
- Service startup types

Tools:
- Task Manager
- Process Explorer

Why It Matters:
- SOC investigations heavily involve process analysis

---

## 6. Basic Troubleshooting

Learn:
- Network troubleshooting
- DNS troubleshooting
- Login issues
- Service failures

Why It Matters:
- Helps differentiate real threats from technical issues

---

# Recommended Tools

- Windows Event Viewer
- Process Explorer
- Task Manager
- Linux Terminal
- Sysinternals

---

# Practical Labs

Practice:
- Create local users
- Check running processes
- Analyze startup applications
- Review event logs
- Kill suspicious processes

---

# Goal Of This Phase

By the end of this phase, you should:
- Understand how systems operate
- Navigate Windows/Linux comfortably
- Understand processes and services
- Use basic command-line tools
